# Workorder Doctrine

A workorder is a durable task contract.

It captures human intent before execution so the project does not depend on chat memory.

Use workorders for substantial, process-sensitive, cross-cutting, or future-facing work.

Do not create a workorder for every tiny edit unless that edit changes future contributor behavior.

The workorder file is the contract, not the AI vendor and not the chat transcript.

See [`../../workorders/README.md`](../../workorders/README.md) and [`../../workorders/TEMPLATE.md`](../../workorders/TEMPLATE.md).
