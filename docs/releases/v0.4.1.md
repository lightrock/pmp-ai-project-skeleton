# DrBones v0.4.1

Release type: documentation, translation, example-routing, and checker-hardening release.

## Summary

`v0.4.1` is a follow-up release after the DrBones public identity release.

The main theme is:

```text
Make the public front door clearer.
Make examples easier to invoke.
Make checks catch drift.
```

## Highlights

- Cleaned up the public `README.md` so it explains DrBones more clearly as a vendor-independent repository template for AI-assisted development.
- Added language links to the main README.
- Added translated README copies under `docs/i18n/`:
  - Spanish: `docs/i18n/README.es.md`
  - French: `docs/i18n/README.fr.md`
  - German: `docs/i18n/README.de.md`
  - Brazilian Portuguese: `docs/i18n/README.pt-BR.md`
  - Hindi: `docs/i18n/README.hi.md`
- Added `AGENTS.md` guidance that `README.md` is the canonical public template README and translations should stay synchronized against it.
- Added a day-in-the-life trigger map to `AGENTS.md` so future foreground AIs know when to read the matching examples.
- Added `examples/day-17-failed-test-repair-loop/README.md` for the failed-test repair loop.
- Updated `tests/test_pmp_check.py` so each `examples/day-NN-short-slug/README.md` must be linked from both `examples/TRIGGER_MAP.md` and `examples/README.md`.
- Updated Day 15 so reference repositories can include architecture repos, method libraries, playbook collections, standards/example repos, or prior projects with reusable patterns.
