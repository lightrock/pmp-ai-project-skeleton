# PMP AI Project Skeleton v0.3.1

Release type: documentation and workflow clarity release.

## Summary

`v0.3.1` tightens the public README and examples around the foreground AI / executor AI workflow.

This release does not change the core governance model from `v0.3.0`. It makes the workflow easier to use in a fresh chat, easier to explain to another human, and easier to hand off to an executor AI without chat noise.

## Highlights

### Useful foreground AI commands added

The README now includes a **Useful commands** section immediately after the Foreground AI quickstart.

It includes copy/paste prompts for:

- starting a new tab against this repository
- making a workorder with preview
- making a workorder quietly
- deciding whether a request belongs in foreground chat or executor work
- running local checks
- reading the day-in-the-life examples

### Workorder preview is now the default human-friendly path

The README now distinguishes two workorder creation modes:

- **Make a workorder with preview** — show the human a clean copy/paste workorder first.
- **Make a workorder quietly** — save the workorder directly and return only a short confirmation plus the repository link.

The preview mode explicitly says the preview block should contain only the workorder body, with no citations, assistant notes, links, explanations, or chat-only commentary inside the copy/paste block.

### Workorders clarified as AI-to-AI task contracts

The Workorders section now says the normal PMP path is:

```text
human intent
→ foreground AI clarifies the task
→ foreground AI writes the workorder
→ executor AI performs the bounded work
→ checks and completion notes tie the result back to intent
```

Workorders can still be written manually, but PMP assumes they are often generated by AI for AI.

### Optional RAG behavior clarified

The Core idea section now clarifies that projects may add retrieval or RAG behavior in whatever way fits their architecture.

PMP does not require a specific vector database, retrieval pipeline, embedding model, indexer, or AI vendor.

### WebGL example discoverability improved

The WebGL Demo 3 README now points directly to the runnable demo file:

```text
examples/webgl-demo-3/webgl_demo3.html
```

It also clarifies that the example is a way to test how smart the repository is: can the AI preserve architecture cognition while producing a concrete artifact?

## Known limitation

This release was prepared through GitHub file operations, not from a local checked-out runtime environment.

The release changes are documentation-only, but the standard local checks should still be run before tagging or publishing a GitHub Release:

```text
python tools/pmp_check.py --area all
python -m pytest
```

If those checks reveal issues, fix them before publishing the GitHub release object.
